В правилах выполнения задач указывается:

■        Тип операции:

o    Погрузка;

o    Разгрузка;

o    Передача документов;

o    Получение документов;

o    Дополнительная операция;

o    Выемка ценностей;

o    Передача ценностей.

■        Перевозчик.

■        Водитель – если водитель не указан, то по умолчанию, правила устанавливаются для всех водителей.

■        Вид адреса - если вид адреса не указан, то по умолчанию, правила устанавливаются для всех видов адресов.

■        Настройки:

o    Требуется сканирование штрихкода (QR-кода);

o    Разрешена установка нового штрихкода (QR -кода);

o    Требуется фотография при выполнении задачи;

o    Требуется фотография при отмене задачи;

o    Необходимость фотографии определяется на уровне проблемы;

o    Запрещено отчитываться о выполнении работы;

o    Способ присоединения внешних файлов, устанавливается документ, к которому будет присоединен файл:

o    К рейсу;

o    К прохождению точки;

o    К заявке на перевозку груза;

o    Способ формирования номера работы, используется только при выполнении работ с точностью до задания на перевозку:

o    Номер из задания на перевозку;

o    Номер КИС из задания на перевозку;

o    Номер из заявки на перевозку;

o    Номер КИС из заявки на перевозку;

o    Выводить комментарии по заявке на перевозку груза;

o    Выводить номер исходного документа:

o    Номер из задания на перевозку;

o    Номер КИС из задания на перевозку;

o    Номер из заявки на перевозку;

o    Номер КИС из заявки на перевозку.

o    Разрешена погрузка/разгрузка с точностью до товарного состава;

o    Требуется отчитываться по каждой строке товарного состава.

Кроме этого, в информационной базе может храниться связанная с партнером информация о подразделениях партнеров, взаимодействиях и соглашениях, которая используется при вводе документов системы.
