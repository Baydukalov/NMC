"РМ: Управление ремонтами"

Рабочее место РМ: Управление ремонтами может также использоваться для создания нового документа Заявка на ремонт (ТО), Ремонт (ТО), но основное его назначение связано с контролем потребности в выполнении технического обслуживания транспортных средств.

Рабочее место состоит из двух списков. В верхнем списке можно просмотреть все транспортные средства или транспортные средства, отобранные по конкретным правилам, с анализом их состояния. При помощи набора кнопок можно отобрать транспортные средств находящиеся в ремонте, ожидающие ремонта и не требующие ремонта соответственно. Более сложные отборы можно настроить в отдельном интерфейсе при помощи кнопки Отбор. Обор по транспортным средствам, относящимся к конкретному гаражу выполняется автоматически после выбора гаража в реквизите Гараж. Кроме этого, текущая потребность в ремонте отображается в отдельной колонке пиктограмм для каждого транспортного средства.

В нижнем списке доступен список документов Заявка на ремонт (ТО), Ремонт (ТО). Просроченные ремонты выделяются в этом списке шрифтом красного цвета. Документы, связанные с транспортным средством, выделенным в верхнем списке по команде Только текущие сортируются в нижнем списке.

При помощи команды Создать на закладке Заявки в информационной базе может быть создан новый документ Заявка на ремонт (ТО) для транспортного средства, выделенного в верхнем списке рабочего места.

При помощи команды Создать на закладке Ремонты в информационной базе может быть создан новый документ Ремонт (ТО) для транспортного средства, выделенного в верхнем списке рабочего места.

Обновить плановые даты ближайших ремонтов и ТО для списка транспортных средств можно при помощи Параметров управления ремонтами,. установив Срок отсчета среднесуточного пробега: за последнюю неделю, за последние две недели, за последний месяц, за последние три месяца, за последние полгода. Установленный срок используется при определении прогнозируемой даты ТО.

В Параметрах управления ремонтами задаются:
  • Сроки отсчета среднесуточного пробега: за последнюю неделю, за последние две недели, за последний месяц, за последние три месяца, за последние полгода.
  • Способ планирования ремонтных работ:
    • По выбору. В табличной части документов доступны две закладки Заявки и Ремонты.
    • Документом Заявка на ремонт (ТО). В табличной части документов доступна закладка Заявки.
    • Документом Ремонт (ТО). В табличной части доступна закладка Ремонты.
  • Настроить оповещения Предупреждать о наступлении срока ТО: за количество дней, количество километров, количество моточасов.

Сроки ближайших ТО определяются по нормам технического обслуживания указанных для модели транспортного средства.
