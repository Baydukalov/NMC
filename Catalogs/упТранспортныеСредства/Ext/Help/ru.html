Справочник «Транспортные средства»

Для обозначения в системе транспортных средств, которые могут использоваться при выполнении рейсов, а также их характеристик, предусмотрен справочник Транспортные средства.

В данном справочнике должны быть заведены все транспортные средства, с которыми работает компания, в том числе собственные, наемные или арендуемые.

В концепции системы в качестве транспортных средств могут рассматриваться реальные транспортные средства, физически выполняющие перевозку грузов, и виртуальные транспортные средства, которые используются для определения обеспеченности перевозки.

 Для удобства работы пользователей при выполнении процедур поиска и выбора транспортных средств в форме списка справочника может быть использован отбор по перевозчику-владельцу транспортных средств.

На основании заданных в справочнике Транспортные средства характеристик в системе в дальнейшем производится планирование процессов, таких как: Планирование рейсов, Назначения ТС на рейсы, Контроль и исполнение рейсов и т.д.

Ведение и организация справочника выполняется на усмотрение Пользователя системы. ТС могут быть сгруппированы, например, по Перевозчикам или по Типам ТС. Для этого необходимо создать отдельную группу.

  • Для каждого ТС указывается Наименование, Регистрационный номер, Идентификационный номер(VIN) и Дата выпуска, где Регистрационный номер в дальнейшем будет заполнятся при формировании комплекта сопроводительных документов по данным ТС, в которые входят ТТН, Путевые листы и др.
      • o    Если используется загрузка данных из системы взимания платы ПЛАТОН, то регистрационный номер транспортного средства необходимо вводить по определенному формату, например, А111АА77 или А111АА177.

  • Вид транспорта - устанавливается вид доступный из перечня:
    • Водный;
    • Воздушный;
    • Легковой автомобиль;
    • Грузовой автомобиль;
    • Спецтехника;
    • Автобус;
    • Железнодорожный;
  • Также для ТС задается принадлежность к конкретному Перевозчику, путем выбора из справочника Партнеров. В дальнейшем используется при подборе ТС перевозчиков при формировании рейсов, назначении ТС на рейсы.       
  • Собственное транспортное средство – указывается признак для собственного транспортного средства.

  • Принято к учету – признак устанавливается у собственных транспортных средств. Документ Принятие к учету транспортного средства формируется перед началом работы в системе с транспортным 
  • Тип ТС – один из основных классификационных признаков транспортного средства, определяющий его назначение и общее конструктивное исполнение. В зависимости от характеристик Типа ТС в системе осуществляется подбор заданий в Рейсы, а именно сопоставляется значение массы и объема, заданные для типа ТС, с суммарной массой и объемом заданий.

  • На уровне Модели ТС задаются уточненные весогабаритные характеристики ТС, Статьи расходов и Нормы расхода топлива;

  • Для ТС может быть задан Индивидуальный график работы, который определяет доступность конкретного ТС в определенный день или временной интервал. График работы используется при планировании ТС в рейс.

На закладке Основных характеристик для Транспортного средства также задаются внешние габариты, на основании которых производится автоматический расчет вместимости ТС. Данные по ВГХ могут наследоваться из установленного Типа ТС или Модели ТС. На основании Объема, Грузоподъемности или Количества грузовых мест ведется учет вместимости грузов по Заданиям на перевозку в ходе планирования рейсов. Выбор параметра расчета вместимости (масса, объем, место или загрузочные метры) задается в Параметрах планирования рейсов - Контролируемый показатель.

При задании Количества загрузочных метров можно рассчитать грузовое пространство, занимаемое грузом по длине ТС. 

Количество посадочных мест задается в случае необходимости учета дополнительных сотрудников при доставке груза, например, грузчиков, которые участвуют в погрузо-погрузочных работах. 

Количество грузчиков указывается в Заявке на перевозку груза, на основании установленного значения при планировании рейсов в РМ: Планирование рейсов подбирается ТС с соответствующим значением.

 Выбор контролируемого параметра зависит от особенностей работы компании-перевозчика.

Например

    При организации перевозки ваты необходимо использовать показатель объема, так как вес данного груза не велик.

    При организации перевозки бумаги для баннеров важно указывать загрузочные метры, так как длина одного рулона бумаги может достигать трех метров и более, соответственно не все транспортные средства имеют указанную длину кузова.

    При организации перевозки продуктов питания, в основном, используется показатель массы груза, для того, чтобы предотвратить перегруз транспортного средства.

    При организации перевозки паллетоместами используется показатель вместимости количества грузовых мест в транспортное средство, где при планировании рейса ведется контроль вместимости грузовых мест в ТС.

    • Основной водитель ТС – указанный водитель используется для автоматического присвоения в рабочем месте Планирование рейсов, Назначение ТС на рейсы;

    • Второй основной водитель ТС – указанный водитель задается как второй водитель ТС в РМ: Назначение ТС на рейсы;

    • Основной экспедитор – указывается сотрудник, который сопровождает грузовую перевозку;

    • В Доступных географических зонах для ТС задаются зоны, в которых возможна перевозка на данном ТС. Если специфическая зона не задана, то реквизит заполняется значением по умолчанию «Доступны все зоны»;

    • Значение Типа кузова может наследоваться из выбранного Типа ТС. Под Типом кузова понимается вариант загрузки грузов в ТС (боковая загрузка, задняя погрузка/выгрузка, верхняя погрузка/выгрузка).

    • Значение Температурного режима может наследоваться из заданного Типа ТС. Задание Температурного режима необходимо в случае установки холодильного агрегата в ТС с целью поддержания определенного температурного режима.
    • Экологический класс – устанавливается из паспорта транспортного средства как дополнительная информация.
    • Категория – устанавливается по техническому регламенту в качестве дополнительной информации.

      • Для транспортных средств могут быть заданы Гаражи.

      Данные используются для формирования точек отправления и точек прибытия в документе Рейс.

      При этом гараж может быть связан с конкретной зоной погрузки или отгрузки. При установке флага Автоматически подбирать гараж по зонам начала/завершения работ в рейсе система при планировании рейса по данным этой закладке позволяет определить и добавить в рейс точки отправления и прибытия.

      Важно!

      Для того чтобы в системе работал механизм автопобора гаража при вводе данных на закладке Гаражи должны выполнятся следующие условия:

      • В одной Зоне погрузки (разгрузки) может быть указан только один гараж.
      • Если необходимо отправление/возвращение ТС в одну точку, то Зону указывать не обязательно.

      На закладке Классы опасности могут быть перечислены классы опасности грузов, транспортировку которых может осуществлять конкретное транспортное средство. Если классы опасности грузов не уточняются, то предполагается что данное транспортное средство может перевозить грузы любого класса опасности. Информация на этой закладке используется в рабочем месте Назначения транспортных средств на рейсы.

      На закладке Изображение может храниться изображение ТС.

      Текущее состояние ГСМ в транспортном средстве, пробега и уровня топлива можно контролировать на закладке Пробег/ГСМ. Значения указанных параметров изменяются автоматически при отражении в системе фактов заправок и сливов ГСМ, расхода по путевым листам, проведения ремонтных работ и т.д. Вручную изменение вышеперечисленных параметров возможно при помощи команды Изменить, расположенной в табличной части и связанной с реквизитом Текущий побег. При изменении пробега или состояния ГСМ будут созданы документы Корректировка пробега ТС и Корректировка остатков ГСМ соответственно.

      Также на этой закладке Пробег/ГСМ указывается следующее:

      • Датчик уровня топлива – указывается датчик уровня топлива, установленный на данное ТС. Данные с датчика используются для автоматического заполнения путевого листа.
      • Счетчик моточасов – при установленном признаке расход ГСМ в Путевом листе учитывается в моточасах.
      • Автоматически определять конечный пробег по данным датчиков – при установленном признаке, конечный пробег определяется автоматически по данным датчиков.
      •   Автоматически определять конечный остаток топлива по данным датчиков - при установленном признаке, конечный остаток определяется автоматически по данным датчиков.

      На закладке Дополнительно указываются следующие параметры транспортного средства:

      • Направление деятельности - указывается направление деятельности Организации. По команде Изменить возможно изменить направление деятельности транспортного средства.
      • Стиль маркера и Стиль линий задает визуальное отображение ТС на картографических сервисах. Выбранные стили маркера и линий можно увидеть в Рабочем месте Спутникового мониторинга.
      • Перерыв между рейсами (в часах) – данный показатель устанавливается у реальных транспортных средств, и учитывается при планировании рейсов в рабочем месте Планирования рейсов. Заданный интервал является страховым запасом для ТС. Планирование ТС в рейс в данный период невозможно;

      Важно!

      В системе могут рассматриваться два вида ТС:

      • Реальные транспортные средства – транспортные средства, с которыми работает компания, в том числе собственные, наемные или арендуемые, а также прицепы с учетом всех характеристик.
      • Виртуальные транспортные средства – транспортные средства, которые помогают планировать рейсы без указания конкретных данных ТС, таких как наименование, регистрационный номер и т.д. Использование виртуальных транспортных средств настраивается в разделе Администрирование - Настройки системы - Транспортные средства и планирование.

      • Печатная форма путевого листа – шаблон путевого листа, используемый по умолчанию при подборе ТС в рейс.
      • Пакет необходимых документов, при помощи которого определяется перечень документов, необходимых для выполнения рейса.
      • Номер кузова, указывается номер кузова ТС. 
      • Номер двигателя, указывается номер двигателя ТС.

      • Подразделение, указывается подразделение компании, которому принадлежит ТС.
      • Запретить фиксировать данные о местоположении с мобильного клиента – при установленном признаке данные о местоположении ТС получаемые с мобильного клиента в системе не учитываются. Например, при установленных трекерах на ТС, данные о местоположении фиксируются именно от них.

        • o    Только при наличии установленных трекеров других систем мониторинга – при установленном признаке выше указанный реквизит теряет своё свойство, сбор данных о местоположении фиксируется с мобильного клиента.

      На закладке Комментарий указывается дополнительная текстовая информация, связанная с эксплуатацией транспортного средства.
